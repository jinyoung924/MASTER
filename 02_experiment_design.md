# 방향 1 실험 설계서 — Inter-Stock 오염 전이 (코딩 에이전트 전달용)

> **문서 성격**: "무엇을/왜 만들어야 하는가"를 규정하는 실험 설계 사양 (3부작 중 2번).
> 구체 코드·환경·파이프라인은 동반 문서 ③ `03_implementation_spec.md` 참조.
> **검증 클레임**: C1(전이 현상), C2(전이 경로=inter-stock attention), C3(중심성 의존, 선택).

---

## 0. 목적과 범위

- **목적**: 사전학습된 MASTER 체크포인트를 **추론 전용**으로 사용해, 일부 종목 입력 오염이 *오염되지 않은* 종목 예측을 저하시키는지(C1), 그 경로가 inter-stock attention인지(C2)를 측정한다.
- **범위 내**: CSI300(주), CSI800(재현 확인). 지표는 IC, RankIC(및 ICIR/RankICIR).
- **범위 외(하지 말 것)**:
  - 모델 **재학습/파인튜닝** 금지 (체크포인트 그대로 사용).
  - AR/IR 등 포트폴리오 지표 금지 (Qlib 백테스트 필요 → 본 프로젝트 비범위).
  - 시장정보 채널(158~220) 오염 금지, 라벨(221) 오염 금지.

## 1. 에이전트가 신뢰해도 되는 베이스라인 사실 (코드로 확인 완료)

| 항목 | 확정된 사실 |
|---|---|
| DataLoader 산출 | 하루치 배치 1개 = 그 날짜의 전 종목. `squeeze` 후 텐서 `(N, T, F)` |
| 차원 | `N≈300(CSI300)/800(CSI800)`, `T=8`, `F=222` |
| 채널 맵 | `[0:158]` 종목 factor(Alpha158) · `[158:221]` 시장정보 63개(전 종목 동일) · `[221]` 라벨 |
| 모델 입력 | `feature = data[:, :, 0:221]` (라벨 제외), `label = data[:, -1, 221]` |
| 모델 내부 | `src = x[:,:,:158]`, `gate_input = x[:,-1,158:221]` (게이팅에 시장정보 사용) |
| inter-stock attention | `SAttention` 모듈. 내부 attention 행렬 `A` 형상 `[T, N, N]` (시점별 종목×종목) |
| 추론 결과 | `predict()`가 일자별 IC/RankIC 계산 후 평균. ICIR=mean/std(over days) |
| 결정성 | 추론은 `eval()` 모드 → dropout off → **입력이 같으면 출력 동일**(반사실 비교 가능) |
| 인덱스 | `dl_test.get_index()` = MultiIndex `(datetime, instrument)`, datetime 정렬 |

## 2. 핵심 실험 아이디어 — 페어드 반사실(paired counterfactual) 비교

이 설계의 심장. 한 날짜 배치에 대해 **두 번 추론**한다.

- **Run-0 (clean)**: 원본 입력 그대로 → 깨끗한 종목들의 예측 `p0`.
- **Run-1 (corrupted)**: 같은 날짜에서 오염 대상 종목들의 factor만 변형 → 깨끗한 종목들의 예측 `p1`.

깨끗한 종목의 입력은 두 Run에서 **완전히 동일**하다. 따라서 `p1 ≠ p0`라면, 그 차이는 오직 *오염된 다른 종목을 거쳐 전파된 것*이다. 이것이 전이의 직접 증거다.

- **종목-독립 모델(LSTM 등)**: `p1 = p0` (수학적으로 불변).
- **MASTER**: inter-stock attention이 있으면 `p1 ≠ p0` 가능.
- **MASTER + self-only attention**: 전이 경로가 끊겨 `p1 = p0` 이어야 함.

> 이 마지막 줄이 C2의 결정적 통제다: **self-only로 바꾸면 깨끗한 종목의 예측 변화량이 수치적으로 0이 되어야 한다.** 안 되면 마스킹 구현이 틀린 것 (→ §10 새너티 체크).

## 3. 실험 조건 매트릭스

각 셀마다 §2의 페어드 추론을 전 test 기간에 대해 수행한다.

| 축 | 값 |
|---|---|
| 모델 조건 | `ATTN_ON`(원본) · `ATTN_SELF_ONLY`(통제군) |
| 오염 비율 `k` | {0, 5, 10, 20, 40} % |
| 오염 유형 | `gaussian` · `missing(zero)` · `shock` (필요 시 `swap`) |
| 오염 강도 | gaussian σ ∈ {0.5, 1.0, 2.0, 3.0} (주 분석은 σ=2.0 고정 후 k 스윕) |
| seed | {0,1,2,3,4} (오염 대상 종목 무작위 추출 재현) |
| 데이터셋 | CSI300(주) · CSI800(재현) |

**스윕 우선순위(작업 순서)**:
1. **메인 곡선**: `gaussian, σ=2.0`, k 스윕, ATTN_ON vs SELF_ONLY, seed 0–4, CSI300.
2. **유형 비교**: k=20% 고정, 오염 유형 비교.
3. **중심성(C3)**: k=10% 고정, 허브 vs 무작위 종목 선택 비교.
4. **재현**: 메인 곡선을 CSI800에서 반복.

## 4. 독립변수 상세

### 4.1 오염 대상 종목 선택 (seed로 재현)
- test에 등장하는 전체 instrument 집합에서 **k%를 무작위로 한 번 뽑아 고정**한다(= "오염 종목 집합", seed별 고정).
- 날짜마다 다시 뽑지 **않는다**. 고정된 집합 중 그 날 존재하는 종목만 오염.
- 평가 대상 "깨끗한 집합" = 오염 종목 집합에 속하지 않는 종목 (전 기간 고정 정의).

### 4.2 오염 연산자 (factor 채널 `[0:158]`에만 적용)
- `gaussian`: 정규화 공간에서 `x += N(0, σ)` 후 `[-3, 3]` 클립(분포 이탈 방지).
- `missing(zero)`: 해당 종목 factor를 0으로 설정(전처리 Fillna 기본값과 동일한 의미의 "결측").
- `shock`: factor를 극단값(예: 부호 무작위 `±3`)으로 치환해 "가짜 급변동" 모사.
- (선택) `swap`: 다른 무작위 종목의 factor 윈도우로 통째 교체.

> 모든 연산자는 **시장정보·라벨·미래 정보 미접촉**, **입력 윈도우 내부만** 변형.

### 4.3 모델 조건
- `ATTN_ON`: 원본 그대로.
- `ATTN_SELF_ONLY`: `SAttention`의 attention 행렬을 단위행렬로 강제(각 종목이 자기 자신만 참조). 구현은 ③에서 forward 패치로 제공.

## 5. 종속변수와 측정

전부 **깨끗한 종목 부분집합**에 대해서만 계산한다.

| 기호 | 정의 | 용도 |
|---|---|---|
| `clean_IC(k)` | 깨끗한 종목에 대한 일자별 Pearson IC의 기간 평균 | C1 주지표 |
| `clean_RankIC(k)` | 동일하나 Spearman | C1 주지표(순위) |
| `ΔP(k)` | 깨끗한 종목의 \|p1 − p0\| 평균(일자·종목 평균) | **C2 직접 지표** |
| `rel_drop(k)` | `1 − clean_IC(k)/clean_IC(0)` (조건별 자기 기준 정규화) | ON vs OFF 공정 비교 |

**왜 `rel_drop`인가**: self-only 모델은 inter-stock 이점을 제거하므로 k=0에서의 절대 IC가 ON보다 낮다. 절대값 비교는 불공정하다. 따라서 **각 조건을 자기 자신의 k=0 대비로 정규화**해 "기울기(=강건성)"를 비교한다.

### 측정 절차(개념)
1. 조건(모델·유형·σ·seed) 고정.
2. 각 날짜: Run-0(clean)으로 `p0`, Run-1(corrupted)으로 `p1` 산출.
3. 그 날의 깨끗한 종목만 골라 `calc_ic(p1, label)` → 일자 IC/RankIC. `mean|p1−p0|` → 일자 ΔP.
4. 전 일자 평균 → `clean_IC(k)`, `ΔP(k)`.
5. seed 0–4 반복 → 평균±표준편차.

## 6. C2 통제 — self-only attention 사양

- `SAttention.forward`에서 softmax attention `A`(형상 `[T,N,N]`)를 **단위행렬 `I_N`**(시점마다 동일)로 치환.
- 효과: 각 종목 표현이 자기 value만 받음 → 종목 간 정보 혼합 완전 차단.
- **기대 동작**: 같은 오염을 줘도 깨끗한 종목의 `p1 == p0` (∴ `ΔP ≈ 0`, `clean_IC` 불변).
- 이 통제로 "k% 제외 때문에 평균이 변한 것 아니냐"는 반박을 제거한다(깨끗한 집합을 ON/OFF에서 동일하게 고정하므로 평가집합 변화 요인이 상쇄).

---

## 7. C3 (선택) — 허브 종목 식별

- `SAttention`의 attention `A[t,i,j]`(i가 j를 참조하는 가중치, 행 합=1)에서 **열 합**(= 종목 j가 받는 총 주목량)을 시점·헤드·전 일자에 걸쳐 누적 → 종목별 **received-attention 중심성**.
- 상위 중심성 종목 = "허브".
- 비교 실험: 동일 k에서 **허브 k개 오염** vs **무작위 k개 오염**의 `clean_IC` 저하 비교. 허브 쪽 저하가 크면 C3 확인.

---

## 8. 산출물

- **표**: `(dataset, model_cond, corrupt_type, σ, k)` → `clean_IC, clean_RankIC, ΔP` 평균±표준편차 (CSV).
- **대표 그림 1**: x=k, y=`rel_drop`(clean_IC 기준), 곡선 2개(ATTN_ON vs SELF_ONLY). → C1·C2 핵심.
- **대표 그림 2**: x=k, y=`ΔP`, 곡선 2개. SELF_ONLY는 0 근처에 붙어야 함.
- **보조 그림**: 오염 유형별 막대, 허브 vs 무작위 곡선.
- **로그**: 각 실행의 seed·조건·소요시간·새너티 체크 결과.

---

## 9. 재현성 요구

- 모든 무작위(오염 종목 추출, gaussian 노이즈)는 seed 고정.
- 추론은 `torch.no_grad()` + `eval()` + `cudnn.deterministic=True`.
- Run-0(clean) 예측은 날짜별 1회 계산 후 **캐시**해 모든 오염 조건에서 재사용(연산 절감 + p0 일관성 보장).
- 결과는 조건별 원자 CSV로 저장해 부분 재실행 가능하게.

## 10. 인수 기준 & 새너티 체크 (구현 전·중 필수)

구현이 옳다는 것을 보증하는 게이트. **하나라도 실패하면 실험 결과 신뢰 불가.**

1. **파이프라인 재현 게이트**: 무오염(k=0) + ATTN_ON으로 `predict` 했을 때, 체크포인트의 공개 성능(`model/performance.xlsx` 또는 README 수치)과 IC/RankIC가 허용오차 내 일치.
2. **self-only 무전이 게이트**: SELF_ONLY 조건에서 임의 k>0 오염 시 깨끗한 종목 `ΔP < 1e-5` (사실상 0). 0이 아니면 마스킹/정렬 버그.
3. **종목-정렬 게이트**: 각 날짜 배치의 `N`이 그 날 instrument 수와 일치(오염/평가 인덱싱이 어긋나지 않음).
4. **채널 불변 게이트**: 오염이 `[158:221]`(시장)·`[221]`(라벨)을 바꾸지 않았음을 텐서 diff로 확인.
5. **단조성 약식 점검**: ATTN_ON에서 k 증가 시 `clean_IC` 비증가 경향(노이즈 허용).
