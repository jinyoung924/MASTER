## 1. 배경: 베이스라인이 주장하는 것

MASTER의 간판 기여는 **inter-stock aggregation**이다. 각 예측 시점에서 모든 종목의 지역 임베딩 \(H^2_t = \|_{u\in\mathcal{S}} h_{u,t}\)를 모아 multi-head attention으로 종목 간 상관을 계산하고, 한 종목의 표현을 다른 종목들의 정보로 보강한다. 논문은 이 메커니즘이 *"각 종목이 순간적으로 상관된 다른 종목들의 신호로부터 이득을 본다"*고 주장하며, 시각화로 그 상관이 실재함을 보인다.

여기서 구조적으로 중요한 사실 하나:

> **한 번의 forward pass(= 한 날짜)에서 모든 종목이 서로를 attend 한다.**
> 따라서 한 종목의 입력 표현은 *같은 날짜의 다른 종목 입력*에 의존한다.

LSTM·GRU·Transformer 같은 종목-독립(per-stock) 모델은 이 의존이 **존재하지 않는다**. 종목 A의 입력을 망가뜨려도 종목 B의 예측은 수학적으로 불변이다. 이 비대칭이 본 프로젝트의 출발점이다.

## 2. 핵심 클레임

### C1 (메인 클레임 — 현상)
**MASTER에서는 일부 종목(k%)의 입력만 오염시켜도, 오염되지 않은 나머지 종목들의 예측 정확도(IC/RankIC)가 함께 저하된다.** 즉 오염이 inter-stock attention을 타고 깨끗한 종목으로 *전이*된다.

- **반증 조건**: 오염 비율 \(k\)를 0 → 40%로 올려도 *깨끗한 종목 부분집합에서 측정한* IC/RankIC가 통계적으로 유의하게 떨어지지 않으면 C1은 기각된다.

### C2 (메커니즘 클레임 — 인과 경로)
**위 저하의 원인 경로는 inter-stock attention이다.** "그냥 좋은 종목 수가 줄어서 평균이 내려간 것"이 아니라, 종목 간 정보 혼합 채널을 통해 오염 신호가 실제로 전파된 결과다.

- **검증 장치**: inter-stock attention을 **self-only(각 종목이 자기 자신만 attend)로 마스킹**한 변형 모델을 통제군으로 둔다. 동일한 오염을 가했을 때
  - attention ON(원본): 깨끗한 종목 IC가 크게 하락
  - attention OFF(마스킹): 깨끗한 종목 IC가 거의 불변
  이 대비가 나타나면 C2 확인.
- **반증 조건**: ON과 OFF에서 깨끗한 종목 IC 하락폭이 동일하면, 전이 경로는 inter-stock이 아니므로 C2 기각(이 경우 저하 원인은 평가 집합 변화 등 다른 요인).

### C3 (스케일링 클레임 — 보조, 선택)
**전이 강도는 (a) 오염 비율 k, (b) 오염 강도, (c) 오염 종목의 "연결 중심성"에 단조적으로 증가한다.** 특히 attention 상으로 많은 종목이 참조하는 *허브 종목*을 오염시키면, 무작위 종목을 오염시킬 때보다 깨끗한 종목 피해가 크다.

- **반증 조건**: 허브 종목 오염과 무작위 종목 오염의 피해가 차이 없으면 C3의 중심성 부분 기각.

> **클레임의 의의**: C1·C2가 성립하면, "종목 간 상관 모델링"이라는 MASTER의 *강점*이 동시에 *강건성 취약점*임을 보이는 것이다. 이는 임의의 모델에 노이즈를 뿌리는 일반적 robustness 실험과 달리, **베이스라인의 고유 설계를 직접 겨냥**한 차별화된 주장이다.

## 3. 검증 설계 개요 (개념 수준)

### 3.1 변수 정의

| 구분 | 변수 | 설명 |
|---|---|---|
| 독립변수 | \(k\) | 오염 종목 비율 ∈ {0, 5, 10, 20, 40}% |
| 독립변수 | 오염 유형 | 가우시안 노이즈 / 특징 결측(0대입) / 가짜 급변동 신호 등 |
| 독립변수 | 오염 강도 | 노이즈 σ 등 (정규화 공간 기준 작은 값부터) |
| 독립변수 | 모델 조건 | {원본 attention ON, self-only 마스킹 OFF} |
| 통제 | seed | 오염 대상 종목 무작위 추출 seed (≥5회 반복) |
| **종속변수** | **clean-IC / clean-RankIC** | **오염되지 않은 (1−k)% 종목들에 대해서만 계산한 IC/RankIC** |

핵심은 종속변수를 **"깨끗한 종목 부분집합"에서만** 계산한다는 점이다. 오염 종목까지 포함해 평가하면 "오염된 종목이 못 맞히는 건 당연" → 클레임 검증이 흐려진다. 깨끗한 종목만 보면 *순수하게 전이된 피해*가 분리된다.

### 3.2 무엇을 오염시키는가 (중요)

- 오염은 **종목 고유 채널(Alpha158 계열)에만** 적용한다.
- **시장 특징(63개) 채널은 건드리지 않는다.** 이 채널은 같은 날짜의 모든 종목에 공유·동일하며 gating(\(m_\tau\))에 쓰이므로, 여기를 건드리면 "종목 단위 오염"이 아니라 "시장 신호 오염"으로 변질된다. (정확한 채널 인덱스/분리 지점은 파일 ③에서 dataloader·모델 코드로 확정)
- **라벨 채널과 미래 정보는 절대 변형하지 않는다** (데이터 누수 방지).
- 오염은 **입력 lookback 윈도우 내부에서만** 일어난다.

### 3.3 절차(공유 추론 루프)

1. 체크포인트 로드 + `*_dl_test.pkl` 로드.
2. 각 날짜 배치에서 무작위 k% 종목 선택 → 해당 종목의 고유 채널만 perturb.
3. (조건별) 원본 모델 / self-only 마스킹 모델로 추론.
4. **깨끗한 종목 부분집합**에서 IC/RankIC 계산.
5. seed 반복 → 평균 ± 표준편차 집계.
6. x축=\(k\)(또는 강도/중심성), y축=clean-IC 곡선 + ON/OFF 대비 그래프 산출.

### 3.4 평가 지표 정의

- **IC**: 일자별 예측값과 실제 수익률의 Pearson 상관을 전 기간 평균.
- **RankIC**: 동일하나 Spearman(순위) 상관. 주가 예측은 "순위 선택"이 목적이라 RankIC가 특히 중요.
- 모두 **clean subset 기준**으로 재정의해 계산한다.
- 통계적 유의성은 seed별 분포에 대한 t-검정(또는 부트스트랩 CI)으로 판단.

## 4. 예상 결과

> 아래 수치는 **가설**이며 측정으로 확정한다. 출발점(무오염 baseline)은 **논문 Table 1이 아니라 레포 `model/performance.xlsx`의 오픈소스 데이터 실측치**를 기준으로 삼는다: CSI300 seed0 기준 **IC ≈ 0.065, RankIC ≈ 0.069** (opensource 체크포인트+데이터). 논문(IC 0.064 / RankIC 0.076)은 회사 비공개 원본 데이터 기준이라 RankIC가 특히 다르므로 비교 대상에서 제외한다.

### 4.1 가설 H1 — clean-IC 단조 하락 (C1)

오염 비율 k가 커질수록 깨끗한 종목의 IC가 단조 감소한다. 예상 곡선(개념):

```
clean-IC
0.07 |●
     |  ●
0.05 |     ●
     |        ●
0.03 |            ●
     +----+----+----+----+----  k(%)
       0    5   10   20   40
```

- 정량적 기대: k=40%에서 clean-IC가 무오염 대비 상대적으로 **30~60% 감소**(예: 0.065 → 0.026~0.046). 정확한 폭은 측정으로.

### 4.2 가설 H2 — ON vs OFF 갈라짐 (C2, 결정적 그림)

같은 오염을 줘도 마스킹 OFF(self-only)는 거의 평평하고, 원본 ON만 떨어진다.

```
clean-IC
0.07 |●───────●───────●   ← OFF(self-only): 거의 불변
     |
0.05 |●
     |    ＼
0.03 |        ＼______●   ← ON(원본): 전이 피해
     +----+----+----+----  k(%)
       0   10   20   40
```

- 이 "벌어지는 두 곡선"이 본 프로젝트의 **대표 결과 그림**이 된다. 두 곡선의 간격 = inter-stock 경로로 전이된 순수 피해량.

### 4.3 가설 H3 — 중심성 효과 (C3)

허브 종목 오염 곡선이 무작위 오염 곡선보다 아래에 위치(같은 k에서 더 큰 피해).

### 4.4 결과 시나리오별 해석 (모두 발표 가치 있음)

| 관측 결과 | 해석 | 클레임 |
|---|---|---|
| H1·H2 모두 성립 | 오염이 inter-stock으로 전이됨이 인과적으로 입증 | **C1·C2 확인 (가장 강한 결과)** |
| H1 성립, H2 불성립 | 저하는 있으나 경로가 attention이 아님 | C1만 확인, C2 기각 → "저하 원인 재조사" 라는 새 발견 |
| H1 불성립 | 깨끗한 종목이 견고함 | **C1 기각** → "MASTER의 inter-stock 설계는 의외로 오염에 견고하다"는 반대 결론(역시 보고 가치) |
| H3까지 성립 | 허브 종목이 단일 취약점 | C3 확인 → "소수 허브 종목 방어가 강건성의 핵심"이라는 실무적 함의 |

핵심: **어느 방향으로 나오든 검증된 명제가 남는다.** 확인되면 취약점 입증, 기각되면 견고성 입증. 이 점이 학부 프로젝트로서 안전한 이유다.

## 5. 타당성 위협(Threats to Validity)과 대응

| 위협 | 내용 | 대응 |
|---|---|---|
| 평가집합 변화 교란 | k% 종목 제외 자체가 IC를 바꿀 수 있음 | clean subset을 **동일하게 고정**하고 ON/OFF만 바꿔 비교(C2 통제가 이 위협을 직접 제거) |
| 정규화 포화 | 특징이 이미 ±3 clip → 큰 노이즈는 saturate | σ를 작은 값부터 스윕, clip 경계 효과를 별도 점검 |
| 데이터 누수 | 라벨/미래 정보 변형 시 결과 무의미 | 입력 윈도우의 비라벨 채널만 perturb |
| 무작위성 | 오염 종목 추출이 결과를 흔듦 | seed ≥5 반복, 평균±표준편차 보고, 유의성 검정 |
| 단일 데이터셋 과적합 | CSI300에만 성립할 수도 | 주 결과는 CSI300, **CSI800로 재현 확인**(robustness of finding) |
| 데이터 일자 커버리지 | 공개 데이터는 하루치에 전 종목의 **약 95%만** 포함(원본은 100%였음) | 페어드 비교에서 동일 종목집합을 clean/corrupt 양쪽에 쓰므로 **상쇄됨**. 단 무오염 IC 재현은 논문이 아니라 `performance.xlsx`(같은 95% 데이터로 측정)에 맞춤 |
| 데이터 소스 불일치 | original/opensource 데이터·체크포인트가 값이 다름 | 체크포인트와 데이터 소스를 **반드시 짝**맞춰 로드(예: opensource ckpt ↔ opensource data). Qlib 기본 데이터(시장지수 구성 다름)는 사용 금지 |
| 지표 범위 한계 | AR/IR 미측정 | 본 프로젝트는 ranking metric으로 한정(논문 핵심 주장도 ranking으로 검증 가능)임을 명시 |

## 6. 성공 기준 & 산출물

**성공 기준** (결과 방향과 무관):
1. clean-IC vs k 곡선이 seed 반복과 함께 재현 가능하게 산출됨.
2. ON vs OFF 통제 비교 그래프가 산출되어 C2를 판정함.
3. CSI300 결과가 CSI800에서도 같은 방향인지 확인됨.
4. 위 결과로 C1·C2(·C3)의 확인/기각이 명확히 진술됨.

**최종 산출물**:
- 오염 비율별 clean-IC/RankIC 표 (평균±표준편차)
- 대표 그림: ON vs OFF 갈라짐 곡선
- 보조 그림: 오염 유형별·중심성별 비교
- 결론: MASTER의 inter-stock 설계가 강건성 측면에서 강점인가 취약점인가에 대한 입증된 진술
